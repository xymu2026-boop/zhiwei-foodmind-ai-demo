# 知味 FoodMind：单店 25 万月销老板经营场景模拟 v0.3

> 本文把“月销约 25 万、外卖约 10%、工作日高于周末、炒菜机标准克重”的客户访谈，扩展成一套可用于 Demo 的单店经营模型。全部门店金额为 S（模拟），公式为 C（计算）。

## 1. 基准门店设定

| 项目 | 模拟值 | 说明 |
|---|---:|---|
| 月标价营业额 | 250,000 元 | 当前访谈基线 |
| 堂食占比 | 90% | 225,000 元 |
| 外卖占比 | 10% | 25,000 元；公开代表性企业样本均值 10.4%，仅作校验 |
| 营业天数 | 30 天 | 月均日标价约 8,333 元 |
| 工作日/周末 | 9,000 / 6,500 元左右 | 按 22 个工作日、8 个周末日正好约 25 万元 |
| 面积 | 120 ㎡ | 模拟商业体门店 |
| 桌数/座位 | 20 桌 / 56 座 | 快餐小炒模型 |
| 在册人数 | 12 人 | 含兼职与高峰支援 |
| 等效全职人数 | 10 FTE | 炒菜机降低传统厨师依赖 |
| 堂食账单 | 约 3,300 单/月 | 日均约 110 单 |
| 堂食客流 | 约 6,260 人/月 | 每单约 1.9 人 |
| 外卖订单 | 约 556 单/月 | 外卖客单约 45 元 |
| 全渠道账单 | 约 3,856 单/月 | 日均约 129 单 |

### 1.1 基准门店的经营特点

- 低客单、工作日用餐为主，周末商圈客流较弱。
- 炒菜机负责标准烹饪，厨师角色更接近机器操作员；原料验收、切配、称重和前处理决定出品差异。
- 堂食是利润主体；外卖用于补充时段和触达顾客，不以大规模补贴换量。
- 完整利润必须同时纳入食材、外卖、包装、支付、人工、租金、能源和总部费用。

| 外卖商家优惠 | -2,000 | -0.8% | 约为外卖 8% |
| 退款、退菜与作废净额 | -1,250 | -0.5% | 全渠道 |
| **净营业收入** | **242,250** | **96.9%** | 进入经营损益的收入口径 |

| **费用合计** | **124,500** | **49.8%** | 含折旧 |

### 2.4 利润结果

| 指标 | 结果 | 解释 |
|---|---:|---|
| 门店经营利润 | 29,356 元 | 订单贡献 - 固定/半固定费用 |
| 经营利润率（对净收入） | 12.1% | 29,356 / 242,250 |
| 经营利润率（对标价营业额） | 11.7% | 29,356 / 250,000 |
| 现金经营结余 | 33,356 元 | 经营利润 + 折旧；未考虑资本开支变化 |
| 食材成本率 | 33.0% | 对标价营业额 |
| 人工成本率 | 23.0% | 对标价营业额 |
| 房租物业率 | 14.0% | 对标价营业额 |

这是一个“炒菜机带来较好标准化、外卖占比较低、当前经营正常”的基准店，因此利润率处于公开样本偏上位置。风险店和异常月必须显著低于此值，不能让 8 家店都一样好。

## 3. 盈亏平衡、安全边际和回收期

### 3.1 盈亏平衡

- 订单变动成本及折让合计：96,144 元。
- 对标价营业额的变动成本率：38.46%。
- 变动贡献率：61.54%。
- 含折旧固定/半固定费用：124,500 元。
- 不含折旧的现金固定支出：120,500 元。

| 指标 | 计算 | 结果 |
|---|---|---:|
| 会计盈亏平衡月销 | 124,500 / 61.54% | 约 202,300 元 |
| 现金盈亏平衡月销 | 120,500 / 61.54% | 约 195,800 元 |
| 当前安全边际 | (250,000 - 202,300) / 250,000 | 约 19.1% |
| 每下降 1 万月销的利润影响 | 10,000 × 61.54% | 约 -6,154 元 |

老板首页不应只说“本月盈利 2.94 万”，还应显示：当前月销最多可下滑约 19%，再往下就进入会计亏损。

### 3.2 新店投资回收

| 初始投入 | 模拟金额 |
|---|---:|
| 装修 | 300,000 |
| 炒菜机及厨房设备 | 180,000 |
| 家具、POS 与前厅设备 | 40,000 |
| 押金及前期租金 | 105,000 |
| 开业物料、证照与培训 | 35,000 |
| 启动营运资金 | 80,000 |
| **初始投入合计** | **740,000** |

- 稳态静态回收期：`740,000 / 33,356 ≈ 22.2 个月`。
- 如果第 1—3 月月销分别为稳态的 60%、75%、90%，前三个月累计现金约亏 1.5 万元。
- 在第 4 月达到稳态后，累计现金流约在第 26 个月转正。

产品里应同时显示“稳态静态回收期”和“考虑爬坡后的累计回收期”，否则会过度乐观。

## 4. 老板核心场景模拟

### 场景 1：早上 9 点，老板今天先看什么

**触发**：系统汇总前一营业日和今天开市前状态。

**老板卡片**：

- 昨日标价营业额 9,380 元，较近 4 个同类型工作日 +4.2%。
- 昨日订单贡献 5,720 元，但外卖贡献率下降 5.8 个百分点。
- 小炒黄牛肉出现 3 次“偏柴”反馈，机器调料偏差正常，优先排查牛肉批次和回温时间。
- 青椒明日建议 42kg，店长合计申报 50kg，预计可减少 8kg 冗余采购。
- 未来 7 天现金安全，不需要推迟供应商付款。

**动作**：老板只处理跨店、超权限和重大成本问题；其余自动分派总部运营、采购和店长。

### 场景 2：流水涨了 4%，利润为什么下降 31%

**基准月**：标价 250,000 元，经营利润 29,356 元。

**活动月**：

| 变化 | 基准月 | 活动月 |
|---|---:|---:|
| 标价营业额 | 250,000 | 260,000 |
| 外卖营业额 | 25,000 | 40,000 |
| 商家优惠 | 6,500 | 15,600 |
| 退款/退菜 | 1,250 | 1,300 |
| 外卖平台费 | 4,500 | 8,800 |
| 包装成本 | 667 | 1,067 |
| 额外活动投放/运营费 | 0 | 2,000 |
| 经营利润 | 29,356 | 约 20,200 |

**判断**：新增 1 万元标价流水主要来自深折扣外卖，商家优惠、平台费和包装多增约 1.38 万元；流水上涨并未带来等比例贡献。

**动作**：暂停亏损单品直降，改为高贡献菜搭配套餐；7 日后比较单均贡献与复购，而不是只看单量。

### 场景 3：牛肉采购价上涨 8%

**事实模拟**：本月牛肉预计采购成本 21,600 元，供应商通知下周期涨价 8%。

- 月成本增量：21,600 × 8% = 1,728 元。
- 若不调价，经营利润从 29,356 降至 27,628 元。
- 对标价营业额利润率从 11.7% 降至 11.1%。

**系统应继续计算三种方案**：

1. 保持售价，接受利润下降。
2. 小炒黄牛肉涨价 2 元，按月销量估算可能覆盖多少成本。
3. 切换供应商或调整部位，但必须同时比较出成率、客诉和退货，不能只比较每公斤报价。

### 场景 4：牛肉实际用量为什么比理论多

| 项目 | 数量 |
|---|---:|
| 理论耗用 | 480kg |
| 实际耗用 | 510kg |
| 已记录收货/前处理报损 | 7kg |
| 员工餐 | 3kg |
| 未解释差异 | 20kg |
| 未解释差异率 | 4.17% |
| 按 43 元/kg 估算损失 | 860 元 |

**排查 SOP**：

1. 核对 BOM 是否使用了正确版本。
2. 核对称重单位、领料和退料是否漏记。
3. 对比收货净重、解冻/回温和切配出成率。
4. 关联炒菜机每次目标与实际食材克重。
5. 若差异集中在单一班组或门店，发起盘点和视频/照片复核。

**关闭条件**：连续 7 天未解释差异率低于 1.5%，且所有报损均有原因与证据。

### 场景 5：系统建议 42kg 青椒，店长申报 50kg

**输入**：菜品销量预测、每份 BOM、现有库存、在途、质量调整和安全库存。

**结果**：

- 人工申报 50kg。
- 原始预测 44kg。
- 扣除可用库存并考虑安全库存后最终建议 42kg。
- 实际耗用 43kg，最终建议准确率约 97.7%。
- 相比人工申报减少 8kg 采购，且仅比实际少 1kg。

**产品价值**：保留店长修正权，但必须记录修正理由，并在事后计算“人工修正是否改善误差”。

### 场景 6：外卖单量上涨，但每单只贡献 0.8 元

**低价菜示例：麻婆豆腐**

- 原价 9.9 元。
- 商家折扣 8%。
- 平台收费按模拟合同计。
- 包装 1.2 元。
- 扣除食材后单份贡献约 0.76 元。

**判断**：这道菜可作为套餐引流，但不适合长期深折扣单卖。系统需要显示菜品在“单点、套餐主菜、套餐搭配”三种角色下的贡献，而不是给菜品一个固定毛利率。

### 场景 7：1000 张券核销 220 张，活动到底好不好

**活动**：满 30 减 5。

| 指标 | 模拟值 |
|---|---:|
| 领取人数 | 1,000 |
| 核销订单 | 220 |
| 核销率 | 22% |
| 活动订单 GMV | 8,360 元 |
| 商家优惠成本 | 1,100 元 |
| 无活动基线预测 | 160 单 |
| 增量订单 | 60 单 |
| 增量 GMV | 2,280 元 |
| 增量贡献率 | 60% |
| 增量贡献（优惠前） | 1,368 元 |
| 扣优惠后的增量贡献 | 268 元 |
| 活动 ROI | 24.4% |

如果错误地把 220 单全部算作活动带来的新单，ROI 会被严重高估。活动页必须同时显示“实际订单”和“无活动基线”。

**后续验收**：60 个增量顾客中，30 日内至少 14 人复购，且复购订单不再依赖同等级优惠。

### 场景 8：工作日午市忙，周末却排了同样的人

**模拟时段结构**：工作日日销约 9,000 元，周末约 6,500 元；午市占 45%，晚市占 40%，其余时段占 15%。

**异常**：周末仍沿用工作日 86 工时排班，实际只需约 74 工时。

- 可减少 12 工时/日。
- 按综合用工成本 30 元/小时，周末两天每周可减少约 720 元无效工时。
- 但炒菜机操作员、收货和店长岗位有最低在岗约束，不能只按人数同比缩减。

**产品动作**：按 30 分钟订单、出餐量、岗位技能和最低配置生成排班建议；店长修正后记录原因，次周比较顾客等待和人力成本。

### 场景 9：房租续约上涨 10%

**变化**：房租物业从 35,000 增至 38,500 元/月。

- 月利润减少 3,500 元，降至 25,856 元。
- 新会计盈亏平衡月销约增加 5,700 元，升至约 208,000 元。
- 按当前贡献率，要覆盖涨租需每天多产生约 190 元标价流水。

**决策不能只看“每天多卖 190 元似乎不难”**，还要看：商场合同年限、递增条款、同店客流趋势、装修残值、搬迁损失和新点位回收期。

### 场景 10：供应商、工资、房租集中到期，下周会不会缺现金

**模拟现金日历**：

| 项目 | 金额 |
|---|---:|
| 当前可用现金 | 182,000 |
| 未来 7 日预计 POS/平台到账 | +62,000 |
| 工资及雇主费用 | -57,500 |
| 房租物业 | -35,000 |
| 供应商到期款 | -48,000 |
| 税费及其他 | -2,000 |
| 7 日后预计现金 | 101,500 |
| 最低现金安全线 | 50,000 |

**结论**：当前安全，但如果平台结算延迟 30,000 元或发生集中退款，安全垫会明显收窄。首页应展示“未来 7/30 日现金缺口”，而不是只展示本月利润。

### 场景 11：要不要开第 9 家店

**基准判断**：

- 稳态月销 250,000 元。
- 稳态月现金经营结余 33,356 元。
- 初始投入 740,000 元。
- 静态回收约 22.2 个月，考虑 3 个月爬坡约 26 个月。

**必须做三档敏感性**：

| 情景 | 稳态月销 | 月现金结余估算 | 回收判断 |
|---|---:|---:|---|
| 乐观 | 280,000 | 约 51,800 | 约 15—18 个月含爬坡 |
| 基准 | 250,000 | 33,356 | 约 26 个月含爬坡 |
| 悲观 | 210,000 | 约 8,700 | 回收期可能超过 7 年，不建议开 |

**老板真正需要的答案**：不是“这个城市客流高”，而是“即使月销比计划少 16%，现金还能不能活下来”。

### 场景 12：8 家店比较，不能只按营业额排名

门店排名建议同时看四个象限：

1. **增长且赚钱**：复制打法。
2. **增长但贡献下降**：检查折扣、外卖和菜品结构。
3. **流水稳定但成本异常**：检查采购、损耗、排班和租金。
4. **流水和利润双降**：进入店长复盘或点位退出评估。

总部运营看跨店异常和规则，店长只看本店可执行动作；老板看利润、现金、风险和需要批准的事项。

## 6. 下一轮数据模拟应新增的表

现有 7 天 Excel 应在下一版增加：

1. `支付与资金账单`：订单号、支付、退款、手续费、交易日、记账日。
2. `外卖订单结算`：原价、顾客支付、商家优惠、平台补贴、平台费、配送、包装、退款、结算净额。
3. `员工排班工时`：岗位、计划/实际工时、时薪、加班、缺勤。
4. `固定费用月表`：房租、物业、水电、营销、维修、折旧、税费、总部分摊。
5. `供应商应付`：采购、差异、账期、到期日和付款状态。
6. `会员与活动`：触达、领券、核销、基线、增量、30 日复购。
7. `门店月度损益`：标价收入到经营利润的完整桥接。
8. `现金日历`：未来 7/30 日到账、应付和最低余额。
9. `开店投资模型`：初始投入、爬坡、现金流和回收月份。

## 7. 校验边界

- 公开行业数据只用于判断模拟值是否离谱，不替代客户真实合同与报表。
- 外卖 18% 收费、支付 0.3%、房租 3.5 万和人工 5.75 万均为本 Demo 的 S（模拟）假设。
- 税费采用简化管理模型，不能用于申报或法定财务判断。
- 新店回收期为未折现现金流模型；正式投资决策还应加入资金成本、残值和关店概率。
- 模拟利润必须能从订单、BOM、采购、平台和费用逐层追溯，不能从目标利润倒填。
